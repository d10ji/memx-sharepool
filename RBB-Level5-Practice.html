Types of Computers

1. Which of the following is the primary distinguishing feature of a supercomputer compared to a mainframe computer?

  1. It supports thousands of simultaneous users for transaction processing
  2. It is optimized for extremely high-speed numerical and scientific computation
  3. It uses only analog signal processing
  4. It has no operating system
Answer

B Supercomputers are built for massive parallel numerical/scientific workloads, while mainframes focus on high-volume, high-reliability transaction processing for many users.

2. A computer that accepts continuously varying (analog) input such as a patient's vital signs, processes it digitally, and displays a digital output — as in an ICU monitor — is classified as a:

  1. Digital computer
  2. Analog computer
  3. Hybrid computer
  4. Mainframe computer
Answer

C Hybrid computers combine analog input/sensing with digital processing and output, a classic example being medical monitoring equipment.

Internet, Email & Web Basics

1. Which protocol is used by an email client to send an outgoing message to a mail server?

  1. POP3
  2. IMAP
  3. SMTP
  4. FTP
Answer

C SMTP (Simple Mail Transfer Protocol) handles outgoing mail delivery; POP3 and IMAP are used only to retrieve mail.

2. Which email retrieval protocol keeps messages synchronized on the server, allowing the same mailbox to be accessed consistently from multiple devices?

  1. SMTP
  2. POP3
  3. IMAP
  4. ICMP
Answer

C IMAP leaves mail on the server and syncs read/folder state across devices, unlike POP3 which typically downloads and removes it.

3. The service that translates a human-readable domain name (e.g., www.rbb.com.np) into its corresponding numeric IP address is:

  1. DHCP
  2. DNS
  3. HTTP
  4. SNMP
Answer

B DNS (Domain Name System) resolves domain names to IP addresses; DHCP instead assigns IP addresses to devices.

  1. Firewall rules
  2. Antivirus software
  3. Biometric access control at the entry door
  4. Data encryption
Answer

C Biometric door access restricts physical entry to premises; the other options protect data/systems logically, not the premises itself.

2. Data centers typically use clean/inert gas fire-suppression systems (e.g., FM-200) instead of water sprinklers mainly because:

  1. Gas is cheaper than water
  2. Gas extinguishes fire without damaging live electronic equipment
  3. Water cannot put out any type of fire

Computer Architecture & Registers

1. Which CPU register holds the memory address of the next instruction to be fetched?

  1. Accumulator
  2. Program Counter (PC)
  3. Instruction Register (IR)
  4. Stack Pointer (SP)
Answer

B The Program Counter tracks the address of the next instruction; the Instruction Register instead holds the instruction currently being executed.

2. The register that temporarily holds data being transferred to or from main memory during a read/write cycle is the:

  1. Program Counter
  2. Memory Data Register (MDR)
  3. Status Register
  4. Instruction Register
Answer

B The MDR (also called Memory Buffer Register) buffers data moving between the CPU and memory during each memory access.

3. In the classic von Neumann architecture, the "von Neumann bottleneck" arises because:

  1. Instructions and data are stored in separate memories with separate buses
  2. Instructions and data share the same memory and the same bus, limiting simultaneous access
  3. There is no memory at all, only registers
  4. The CPU has no control unit
Answer

B A single shared memory and bus for both instructions and data (unlike the Harvard architecture) limits throughput between CPU and memory.

Memory Management

1. Which memory management technique divides a process into fixed-size blocks that can be loaded into any available frame in physical memory, eliminating external fragmentation?

  1. Segmentation
  2. Paging
  3. Swapping
  4. Overlaying
Answer

B Paging splits both memory and processes into fixed-size units (pages/frames), so pages need not be contiguous in physical memory.

2. A condition in which the CPU spends most of its time swapping pages in and out of memory rather than executing useful instructions is called:

  1. Deadlock
  2. Starvation
  3. Thrashing
  4. Fragmentation
Answer

C Thrashing occurs when insufficient physical memory forces excessive paging activity, drastically reducing useful CPU work.

3. Virtual memory allows a process to use more address space than physically available RAM chiefly by:

  1. Compressing all data before storing it in RAM
  2. Using a portion of disk storage as an extension of RAM, transferred via paging
  3. Adding an extra physical RAM module automatically
  4. Storing data only in CPU registers
Answer

B Virtual memory maps disk space as if it were extra RAM, moving pages between disk and memory as needed.

Hard-Disk Organization

1. The time required for the disk read/write head to move to the correct track is called:

  1. Rotational latency
  2. Seek time
  3. Transfer time
  4. Burst time
Answer

B Seek time is the head's movement delay; rotational latency is the additional wait for the target sector to rotate under the head.

2. A set of tracks located at the same position across all platters of a hard disk stack is called a:

  1. Sector
  2. Cylinder
  3. Cluster
  4. Partition
Answer

B A cylinder is the collection of identically-positioned tracks on every platter, accessible without moving the read/write heads further.

CPU Architecture & I/O Management

1. Which I/O technique allows a peripheral device to transfer data directly to or from main memory with minimal CPU involvement, freeing the CPU for other tasks?

  1. Programmed I/O
  2. Polling
  3. Direct Memory Access (DMA)
  4. Interrupt-driven I/O only
Answer

C DMA lets a controller move data straight between device and memory over the bus, bypassing the CPU for the actual data transfer.

2. Pipelining improves CPU throughput mainly by:

  1. Increasing the clock speed of the CPU
  2. Overlapping the fetch, decode and execute stages of multiple instructions simultaneously
  3. Adding more cache memory
  4. Reducing the instruction set size
Answer

B Pipelining overlaps different instruction-processing stages across successive instructions, increasing instruction throughput without raising clock speed.

Error Detection & Correction

1. Which error-detection technique, based on polynomial division of the transmitted data, is widely used in Ethernet frames and is highly effective at detecting burst errors?

  1. Parity check
  2. Checksum
  3. Cyclic Redundancy Check (CRC)
  4. Hamming code
Answer

C CRC treats data as a polynomial and divides it by a generator polynomial, giving strong burst-error detection used in the Ethernet FCS field.

2. Unlike a simple parity bit, which can only detect an error, a Hamming code is significant because it can:

  1. Compress transmitted data
  2. Detect and correct single-bit errors
  3. Encrypt transmitted data
  4. Only work on analog signals
Answer

B Hamming codes add multiple parity bits at calculated positions so that a single-bit error can be located and automatically corrected.

Process Management & CPU Scheduling

1. In First-Come-First-Served (FCFS) CPU scheduling, a long process arriving before several short processes causes which problem?

  1. Starvation of high-priority processes
  2. Thrashing due to excessive paging
  3. Convoy effect — short processes wait behind the long process that arrived first
  4. Deadlock among the waiting processes
Answer

C Because FCFS is non-preemptive, one long process at the head of the queue delays all shorter processes behind it.

2. What key advantage does Shortest Job First (SJF) scheduling offer over FCFS?

  1. It always prevents starvation of long processes
  2. It gives the minimum average waiting time among non-preemptive algorithms when burst times are known
  3. It requires no estimate of CPU burst time
  4. It assigns a fixed time quantum to every process
Answer

B SJF is provably optimal for minimizing average waiting time, though it needs advance knowledge (or estimation) of burst times.

3. In Round Robin CPU scheduling, each process is given a fixed slice of CPU time known as a...?

  1. Time quantum — a fixed small unit of CPU time given to each process in turn
  2. Priority level — a number indicating process importance
  3. Burst cycle — the total CPU time a process needs
  4. Dispatch latency — time to switch from kernel to user mode
Answer

A Round Robin cycles through the ready queue, giving each process one time quantum before preempting it, favoring responsiveness.

4. Priority scheduling can cause indefinite postponement (starvation) of low-priority processes. Which technique resolves this?

  1. Reducing the time quantum for all processes
  2. Switching permanently to FCFS scheduling
  3. Disabling all hardware interrupts
  4. Aging — gradually increasing the priority of processes that wait too long
Answer

D Aging raises a waiting process's priority over time so it eventually gets scheduled, preventing indefinite starvation.

Short Answer — Financial Institutions and Key Banking Terms (5 marks each)

1. [5 marks] How are banks and financial institutions (BFIs) classified in Nepal under BAFIA 2073?

Model answer — key points
  • BAFIA 2073 (Banks and Financial Institutions Act) is the umbrella law under which NRB licenses and regulates all BFIs.
  • Institutions are grouped into four classes by function and minimum capital:
  • Class "A" — Commercial banks (banijya bank): full-scope banking, largest capital; e.g., RBB.
  • Class "B" — Development banks (bikas bank): limited scope, often regional.
  • Class "C" — Finance companies: deposits, loans, hire-purchase.
  • Class "D" — Microfinance institutions (laghubitta): small loans to low-income/rural groups for financial inclusion.
  • NRB sits above all four as regulator; cooperatives and the infrastructure development bank operate under separate laws.

2. [5 marks] What is capital fund? Distinguish between core capital and supplementary capital.

Model answer — key points
  • Capital fund (puji kosh) = a bank's own permanent resources that absorb losses and protect depositors; the basis of capital adequacy.
  • Core capital (Tier 1 / primary): paid-up capital, share premium, statutory general reserve, retained earnings — permanent and freely available.
  • Supplementary capital (Tier 2): general loan-loss provision, subordinated term debt, revaluation reserve, hybrid instruments — limited loss-absorbing capacity.
  • Capital Adequacy Ratio (CAR) = Capital fund ÷ Risk-Weighted Assets, computed under Basel III.
  • NRB fixes a minimum CAR and core-capital ratio for each class of BFI (check current figure from NRB).
  • A larger capital fund means a stronger cushion, more public confidence and greater lending capacity.

3. [5 marks] What is the Cash Reserve Ratio (CRR)? How does it differ from the Statutory Liquidity Ratio (SLR)?

Model answer — key points
  • CRR = minimum percentage of total deposits a bank must keep as a cash balance with NRB.
  • Purpose: protects depositors, ensures liquidity, and acts as a monetary-policy tool to control money supply and credit.
  • SLR = minimum share of deposits held in liquid assets (cash, gold, government securities) — a broader requirement that includes the CRR portion.
  • Raising CRR/SLR tightens liquidity (less money to lend); lowering them eases credit.
  • Both ratios are set by NRB in the annual Monetary Policy (check current figure from NRB).

4. [5 marks] Explain the Credit-to-Deposit (CD) ratio and its importance.

Model answer — key points
  • CD ratio = total credit (loans) ÷ total deposits, expressed as a percentage.
  • It shows how much of the deposits a bank has lent out — a key liquidity and prudence indicator.
  • NRB caps the CD ratio to prevent over-lending and liquidity stress (check current figure from NRB); it replaced the earlier CCD ratio.
  • A ratio above the limit signals aggressive lending and liquidity risk; a very low ratio means idle funds and weak earnings.
  • Helps NRB monitor systemic liquidity and forces banks to balance lending with deposit mobilisation.

5. [5 marks] What is the base rate of a bank? Why does NRB require it?

Model answer — key points
  • Base rate = the minimum interest rate below which a bank generally cannot lend, so that lending covers its costs.
  • Computed from an NRB-prescribed formula combining: cost of funds (deposit interest), cost of CRR and SLR, operating cost, and a small return on assets.
  • Actual lending rate = base rate + a premium reflecting the borrower's risk.
  • Purpose: transparency in loan pricing, fair competition, and protection of banks from lending below cost.
  • Banks calculate and publish the base rate monthly (check current figure from NRB).

1. [5 marks] What is loan-loss provisioning? How are loans classified by NRB?

Model answer — key points
  • Provisioning = setting aside part of profit as a reserve against possible loan defaults; it protects the bank and reflects true asset quality.
  • NRB classifies loans by overdue period into: Pass, Watchlist, Substandard, Doubtful, and Loss.
  • Performing loans = Pass + Watchlist; Non-Performing Loans (NPL) = Substandard + Doubtful + Loss.
  • The provision rate rises with risk — smallest for Pass, up to 100% for Loss loans (check current provisioning rates from NRB).
  • Higher NPL and provisioning cut into profit; the NPL ratio is a key measure of asset quality.

2. [5 marks] What is KYC and why is it important in banking?

Model answer — key points
  • Know Your Customer (KYC) = the process of identifying and verifying a customer's identity, address and source of funds before and during a banking relationship.
  • Documents: citizenship/passport, photo, PAN, address proof; for firms — registration papers and beneficial-owner details.
  • Purpose: prevent money laundering, terrorist financing, fraud and misuse of accounts.
  • Part of wider Customer Due Diligence (CDD) required under the AML Act 2064; NRB mandates periodic KYC updates and enhanced KYC for high-risk customers.
  • Foundation of a safe banking relationship and of regulatory compliance.

3. [5 marks] What is blacklisting in banking? Who maintains it and on what grounds?

Model answer — key points
  • Blacklisting = an official listing of defaulters/wrongdoers who are then barred from further credit facilities.
  • Maintained in Nepal by the Credit Information Bureau — Credit Information Center Ltd (Karja Suchana Kendra) — under NRB direction.
  • Common grounds: wilful loan default, misuse of a loan, cheque dishonour due to insufficient funds, and fraud.
  • Consequences: no new loans from any BFI, a damaged credit record, and possible legal action.
  • Removal follows clearing of dues and a cooling-off period; the system enforces credit discipline and protects the banking sector.

4. [5 marks] What is credit rating? Name the credit-rating agencies operating in Nepal.

Model answer — key points
  • Credit rating = an independent expert opinion on the creditworthiness (ability to repay) of a borrower, company or debt instrument.
  • Expressed as grades/symbols, e.g., AAA (highest) down to D (default).
  • Provided in Nepal by licensed agencies such as ICRA Nepal and CARE Ratings Nepal.
  • Mandatory for public share issues, debentures and large borrowings.
  • Helps investors and lenders price risk — a higher rating lowers borrowing cost. (Different from the internal credit scoring of retail borrowers.)

1. [10 marks] Explain motivation, communication and team building as elements of organizational behaviour, and how they apply in a bank.

  • Organizational behaviour (OB) studies how individuals and groups act at work; good OB raises productivity, service quality and staff retention — vital in a service business like banking.

  • Maslow's hierarchy of needs (physiological → safety → social → esteem → self-actualisation).

  • Communication — sharing information to build common understanding.
  • Process: sender → message → channel → receiver → feedback.

  • Barriers (language, noise, hierarchy, overload) are overcome by clarity, feedback and active listening.
  • Team building — developing a cohesive, goal-focused group; Tuckman's stages: forming, storming, norming, performing (and adjourning).
  • Needs shared goals, clear roles, trust and good leadership; benefits branch coordination, problem-solving and customer service.
  • Together, motivated staff, clear communication and strong teams drive service quality and organisational goals.

1. [10 marks] What is Blockchain technology? Explain its key features and describe two potential applications in Nepal's banking and financial sector.

Unit 2: Computer Architecture, Memory, CPU & I/O — Short Answer Questions

1. [5 marks] Draw and explain the memory hierarchy of a computer system.

Model answer — key points
  • Draw as a pyramid, top to bottom: CPU registers → Cache (L1/L2/L3) → Main memory (RAM) → Secondary storage (HDD/SSD) → Tertiary/offline storage (tape, cloud backup).
  • Moving down the pyramid: capacity increases, cost per bit decreases, access speed decreases.
  • Moving up: speed increases, capacity decreases, cost per bit increases.
  • Purpose: bridges the speed gap between a fast CPU and slower storage, exploiting temporal and spatial locality through caching.

2. [5 marks] Explain any three techniques of I/O data transfer used in a computer system.

Model answer — key points
  • Programmed I/O: CPU polls device status continuously and performs the transfer itself; simple but wastes CPU cycles (busy-waiting).
  • Interrupt-driven I/O: device raises an interrupt when ready; CPU runs an ISR to transfer data; CPU is free to do other work between interrupts.
  • DMA (Direct Memory Access): a dedicated DMA controller transfers data directly between device and memory without per-byte CPU involvement; CPU only initiates the transfer and is interrupted on completion; used for bulk transfers (disk, network).
  • Efficiency increases and CPU overhead decreases in the order: programmed I/O → interrupt-driven I/O → DMA.

Unit 2: Computer Architecture, Memory, CPU & I/O — Long Answer Questions

1. [10 marks] Explain the internal organization of a CPU with a block diagram. Describe the fetch-decode-execute cycle and the function of key registers.

Model answer — key points
  • Block diagram: ALU, Control Unit, and Registers connected internally, linked to Memory and I/O via the system bus (address bus, data bus, control bus).
  • Key registers: PC (Program Counter — address of next instruction), MAR (Memory Address Register), MBR/MDR (Memory Buffer/Data Register), IR (Instruction Register — holds current instruction), Accumulator/GPRs (operands and results), Flag/Status register (condition codes).
  • Fetch: PC → MAR → memory read → MBR → IR; PC is incremented.
  • Decode: Control Unit decodes the opcode in IR and generates control signals.
  • Execute: ALU performs the operation or a memory/I/O access occurs; result is written back to a register or memory.
  • Cycle repeats, typically followed by an interrupt-check step; mention pipelining as a performance enhancement that overlaps these stages for successive instructions.

Unit 3: Networking Devices, Switching & Cryptography — Short Answer Questions

1. [5 marks] Differentiate between hub, switch, router, and gateway.

Model answer — key points
  • Hub: Physical layer (Layer 1) device; broadcasts an incoming signal to all ports; no intelligence; single collision domain.
  • Switch: Data Link layer (Layer 2) device; forwards frames only to the destination port using a MAC address table; separate collision domain per port, single broadcast domain.
  • Router: Network layer (Layer 3) device; forwards packets between different networks using IP addresses and routing tables/protocols; separates broadcast domains.
  • Gateway: connects networks using different protocols/architectures (protocol translation), e.g. connecting a bank's LAN to a payment-switch network with a different message format.

2. [5 marks] Differentiate between symmetric key and asymmetric key cryptography, with one example algorithm each.

Model answer — key points
  • Symmetric key: same key for encryption and decryption; fast, suited to bulk data; key-distribution is the main challenge. Example: AES, DES/3DES.
  • Asymmetric key: key pair (public + private); public key encrypts/verifies, private key decrypts/signs; solves key-distribution but is computationally slower. Example: RSA, ECC, Diffie-Hellman (key exchange).
  • Banking use: asymmetric for digital signatures, SSL/TLS handshake, and PKI-based certificate authentication; symmetric for actual bulk encryption of transaction data.
  • Hybrid model: TLS uses asymmetric cryptography to securely exchange a symmetric session key, then uses that symmetric key for the session's data encryption.

Unit 3: Networking Devices, Switching & Cryptography — Long Answer Questions

1. [10 marks] Draw the OSI reference model and explain the function of each layer. How does it compare with the TCP/IP model?

Model answer — key points
  • Draw 7 stacked layers, sender and receiver side, with peer-to-peer logical arrows and the actual data path flowing down and across: Application, Presentation, Session, Transport, Network, Data Link, Physical.
  • Application: interface to end-user services (HTTP, FTP, SMTP).
  • Presentation: data translation, compression, encryption/decryption formatting.
  • Session: establishes, manages, and terminates sessions between hosts.
  • Transport: end-to-end delivery, segmentation, flow and error control, port addressing (TCP/UDP).
  • Network: logical (IP) addressing and routing between networks.
  • Data Link: framing, MAC addressing, error detection (CRC/parity) and flow control over a single link.
  • Physical: transmission of raw bits over the physical medium.
  • TCP/IP model has 4 layers: Application (merges OSI Application+Presentation+Session), Transport, Internet (=Network/IP), Network Access (merges Data Link+Physical).
  • Comparison: OSI is a conceptual, protocol-independent reference model developed before protocols existed; TCP/IP is a practical model built around the protocols actually used on the Internet.

Unit 4: Operating Systems & Scheduling — Short Answer Questions

1. [5 marks] Differentiate between a process and a thread.

Model answer — key points
  • Process: independent program in execution with its own memory space (code, data, heap, stack) and PCB; heavyweight; inter-process communication needs IPC (pipes, shared memory, message queues).
  • Thread: lightweight unit of execution within a process; shares code/data/heap with sibling threads but has its own stack, registers, and program counter; cheaper context switch than a process.
  • Multi-threading benefit: concurrency and responsiveness, e.g. a core banking application server handling many client request threads with shared resources.

2. [5 marks] Differentiate between paging and segmentation as memory management techniques.

Model answer — key points
  • Paging: physical memory divided into fixed-size frames, logical memory into equal-size pages; mapped via a page table; eliminates external fragmentation but may cause internal fragmentation.
  • Segmentation: program divided into variable-size logical segments (code, data, stack) reflecting program structure; mapped via a segment table (base + limit); may cause external fragmentation.
  • Address translation: paging uses page-number + offset; segmentation uses segment-number + offset with bounds checking.
  • Modern OS often combine both — segments further divided into pages.

Unit 4: Operating Systems & Scheduling — Long Answer Questions

1. [10 marks] Explain FCFS, SJF, Round Robin, and Priority CPU scheduling algorithms with a suitable example. What criteria are used to evaluate a scheduling algorithm?

Model answer — key points
  • Evaluation criteria: CPU utilization, throughput, turnaround time, waiting time, response time.
  • FCFS (First Come First Served): non-preemptive, executes in arrival order; simple but suffers from the convoy effect.
  • SJF (Shortest Job First): process with smallest burst time runs next; preemptive form is SRTF; minimizes average waiting time but risks starvation of long processes and needs burst-time estimation.
  • Round Robin: preemptive, each process gets a fixed time quantum in cyclic order; good response time for interactive systems; too small a quantum increases context-switch overhead, too large behaves like FCFS.
  • Priority Scheduling: highest-priority process runs first; preemptive or non-preemptive; risk of starvation for low-priority jobs, mitigated by aging.
  • Illustrate with a small Gantt chart for 3–4 processes (arrival time, burst time) and compute average waiting/turnaround time under each algorithm for comparison.

Unit 5: DBMS, Data Warehousing, Web Technologies & Disaster Recovery — Short Answer Questions

1. [5 marks] Explain the ACID properties of a database transaction. Why are they important for banking systems?

Model answer — key points
  • Atomicity: transaction executes fully or not at all — e.g. a fund transfer's debit and credit both succeed or both roll back.
  • Consistency: transaction moves the database from one valid state to another, preserving integrity constraints.
  • Isolation: concurrent transactions do not interfere; isolation levels — read uncommitted, read committed, repeatable read, serializable.
  • Durability: once committed, changes survive system failure (via write-ahead logging).
  • Importance in banking: guarantees correctness of financial transactions, prevents double-debit/credit errors, and enables crash recovery in Core Banking Systems.

2. [5 marks] Differentiate between OLTP and OLAP systems. What is a data warehouse?

Model answer — key points
  • OLTP (Online Transaction Processing): day-to-day operational transactions (deposits, withdrawals); normalized schema; fast, short transactions; many concurrent users.
  • OLAP (Online Analytical Processing): complex analytical queries and reporting for decision-making; denormalized dimensional schema (star/snowflake); historical, aggregated data.
  • Data warehouse: a subject-oriented, integrated, time-variant, non-volatile collection of data from multiple sources, used for reporting and analysis.
  • ETL: Extract data from source systems, Transform (cleanse/standardize), Load into the warehouse.
  • Banking use: MIS and regulatory reporting to NRB, fraud-pattern analysis, customer segmentation.

3. [5 marks] Differentiate between HTML and XML.

Model answer — key points
  • HTML: HyperText Markup Language; used to display and format web content; fixed, predefined tag set; primarily concerned with presentation.
  • XML: eXtensible Markup Language; used to store, describe, and transport structured data; user-defined custom tags; case-sensitive and strictly well-formed (every tag must be properly closed and nested); concerned with data, not presentation.
  • Relation: XML is commonly used for data interchange between systems (e.g. web-service messaging), while HTML renders content in a browser; XHTML combines the strict syntax rules of XML with HTML tags.

Unit 5: DBMS, Data Warehousing, Web Technologies & Disaster Recovery — Long Answer Questions

1. [10 marks] What is a Disaster Recovery Plan (DRP)? Explain its key components and describe how RTO and RPO are used in designing one for a bank's IT infrastructure.

Model answer — key points
  • DRP: a documented, structured approach describing how an organization resumes operations quickly after an unplanned incident (natural disaster, cyberattack, hardware failure).
  • Key components: risk assessment and Business Impact Analysis (BIA), backup strategy (full/incremental/differential), backup site type, data replication, defined recovery procedures and roles, communication plan, periodic testing/drills.
  • RTO (Recovery Time Objective): maximum acceptable time to restore a system/service after disruption.
  • RPO (Recovery Point Objective): maximum acceptable data loss, measured as a point in time (how far back the last usable backup/replication point can be).
  • Site types: Hot site (near real-time replication, low RTO/RPO, high cost), Warm site (partially equipped, moderate RTO), Cold site (basic infrastructure only, high RTO, low cost).
  • Bank application: Core Banking System typically requires near-zero RPO/RTO, real-time data replication to a DR site in a different seismic/geographic zone, and periodic DR switchover drills — consistent with NRB directives on business continuity.

1. [5 marks] Explain any four common types of cyber-attacks faced by banking and financial institutions.
